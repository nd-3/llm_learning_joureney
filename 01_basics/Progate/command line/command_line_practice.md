**コマンドライン**

# コマンド

```
// ファイル作成
touch practice.txt
```

```
// ファイルの中身を表示
cat practice.txt
```

```
// ディレクトリを作成
mkdir practice
```

```
// ディレクトリ移動
cd practice
```

```
// カレントディレクトリを確認する-->(print working directory)
pwd
```

```
// ディレクトリの中身を確認する-->list
ls
```

```
// 一つ親のディレクトリに移動
cd ..
```

```
// ホームディレクトリに移動
cd
```

```
// ファイルを移動させる
mv practice.txt(移動させたいファイル名) practice2(移動先ディレクトリ)
```

```
// ファイル名を変更する
mv (現在のファイル・ディレクトリ名) (新しいファイル・ディレクトリ名)
```

```
// ファイルをコピーする
cp (コピーするファイル名) (新しいファイル名)
```

```
// ディレクトリごとコピーする
cp  -r (コピーするディレクトリ名) (新しいディレクトリ名)
```

```
// ファイルを削除する
rm (削除するファイル名)
```

```
// ディレクトリを削除する
rm -r (削除するファイル名)
```

# コマンドライン

- コマンドラインには保管機能がある。ファイル名やフォルダ名の入力時に Tab で補完してくれる

* **ファイル構造**を意識することが大切。

* コマンドラインでは自分が今いる場所というものがある。作成したいファイルは作成したいディレクトリに移動する必要がある。自分が今いる場所を**カレントディレクトリ**という。

* 一番親の階層をルートディレクトリという。ルートディレクトリは **/** で表される。

* ホームディレクトリはユーザーが作業するときに起点にするディレクトリのこと。

* 過去に実行したコマンドは **↑** で履歴が残る
