# Webシステムの基本構造

## Webシステムとは

- 離れた場所でもPC同士が会話するプログラム
- お願いする側(ブラウザ)⇔お願いを受けて動く側(サーバ)⇔記録を保管する側(DB)

## CLIとGUIの違い

CLIはキーボード操作
GUIはマウスなどのクリック操作で進める

## 自分のQA Botではどうなってるか

1. CLIで質問を受け取る
2. main.pyで受け取った内容をchromaDBに渡す
3. chromaDBで受け取った内容と関係ありそうな文章の断片を検索して返す(だけ)
4. main.pyで受け取り、Claudeに渡す
5. Claudeが受け取った内容をもとに回答を生成
6. main.pyがCLIに出力

## WSLを使う理由

- WSLとはWindowsの中でLinuxを動かせる仕組み。Windowsの使いやすさを享受しつつ、Linuxの開発環境にできる。
- 本番環境でLinuxを使用していることが多い。開発と本番でOS違いによるバグを防げる。
- Linuxは開発において、OSの介入が少なく、安定して長時間稼働させておける
- 必要性が低い、GUIや使いやすさを省き、軽量
