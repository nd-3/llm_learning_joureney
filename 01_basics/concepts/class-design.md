# クラス設計の基本

```py
class Person: # Personというクラス
    def __init__(self, name, age):
      self.name = name
      self.age = age

    def introduce(self):
      return f"{self.name}です。{self.age}歳です。"

tanaka = Person("田中", 32)   # 田中というインスタンス
print(tanaka.introduce())
```

## クラスとインスタンス

- クラスは設計図。型・設計図だけで、実体はなにも存在していない。
- インスタンスは設計図から作られる実体。クラスからデータをもった実体。

## メンバ変数(self.○○)とは

- `__init__`はコンストラクタと呼称され、クラスから**インスタンスが作られる瞬間**に、自動的に1回だけ実行される関数。
- `self.name = name`は渡されたname(材料)を、このインスタンス自身(self)の持ち物として保持するということ。これがないと渡された材料はその場限りで消えて、後から`tanaka.name`のように呼び出せなくなる。

## dictではなくクラスを使う理由

- スキーマはクラスで決めた型のルールそのもの。データの設計図で構造の取り決め一式のこと。
- 辞書だと打ち間違いや型違いでもエラーにならず、気づかずに事故に繋がる。

## 自分のプロジェクトではどこで使われているか

- `Pydantic`のクラスを使うのは、LLMの出力がスキーマ通りでない場合に、その場で検出できる。AIの出力を信頼できる形にする目的
- `Person`というクラスが名前と年齢のインスタンス変数を持っているのと同じように、実際のプロジェクトの`SkillMatchAnalysis`クラスは点数と説明文のインスタンス変数を持っている。ただし`SkillMatchAnalysis`は、`Pydantic`という仕組み(`BaseModel`)を土台にして作られたクラスなので、点数や説明文におかしな値が来た場合、自動でエラーを出してくれる。
